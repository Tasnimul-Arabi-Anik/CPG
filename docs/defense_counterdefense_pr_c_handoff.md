# Defense/counter-defense evidence handoff

## Scope

This branch is PR C for the defense/counter-defense evidence layer. It is stacked on PR B (`prB-h1-receptor-benchmark`). It wires accepted host defense and explicit phage anti-defense candidate evidence into the production profile while keeping H4 blocked because productive-infection, plaque, propagation, and EOP outcomes are absent.

This PR adds evidence tables and conservative integration behavior only. It does not add a defense/counter-defense predictive model.

## Config wiring

`config/workflow.production.yaml` now supplies:

- `inputs.host_defense_input: data/metadata/production_evidence/host_defense_systems.tsv`
- `inputs.phage_antidefense_input: data/metadata/production_evidence/phage_antidefense_candidates.tsv`

## Integration behavior

`scripts/05_integrate_defense_counterdefense.py` now keeps annotation-keyword phage anti-defense hits as screening-only review candidates. These rows are excluded from compatibility matching unless the same candidate is supplied through an explicit reviewed anti-defense table.

This prevents generic annotation keywords from being treated as accepted counter-defense evidence.

## Reproduction commands

Host DefenseFinder evidence was generated by the project glue script around the established DefenseFinder tool:

```bash
python scripts/build_phagehostlearn_host_defense_evidence.py   --defensefinder-command defense-finder   --work-dir results/production/host_defense/defensefinder   --output data/metadata/production_evidence/host_defense_systems.tsv   --report-output data/metadata/production_evidence/host_defense_systems_report.tsv
```

The current local environment does not have `defense-finder` on `PATH`, so this PR validates the reviewed tracked table and does not rerun DefenseFinder.

Phold ACR anti-defense evidence was regenerated locally from existing Phold outputs:

```bash
python scripts/build_phold_acr_antidefense_evidence.py   --phold-root results/production/phold_assay_phages   --annotations data/metadata/production_evidence/phagehostlearn_prodigal_cds_annotations.tsv   --output data/metadata/production_evidence/phage_antidefense_candidates.tsv   --report-output data/metadata/production_evidence/phage_antidefense_candidates_report.tsv
```

Result: 7 accepted Phold ACR anti-CRISPR candidates across 7 phages.

## Claim boundary

Allowed wording for this branch:

> Production host-defense and explicit phage anti-defense candidate evidence are available for coverage auditing and future defense/counter-defense compatibility tests.

Not allowed from this branch:

- host defense explains infection failure;
- phage anti-defense candidates confer defense escape;
- defense/counter-defense features improve host-range prediction;
- spot-test positives are productive infection;
- H4 is supported.

H4 remains blocked until observed productive-infection, plaque, propagation, or EOP outcomes are curated and a leakage-safe paired analysis is defined.
