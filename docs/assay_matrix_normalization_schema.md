# Assay Matrix Normalization Schema

`scripts/normalize_assay_matrix.py` converts reviewed host-by-phage interaction matrices into the canonical assay source export consumed by `scripts/import_phage_host_assays.py`.

This layer is source-specific curation support. It does not download data and does not make biological claims. Blank matrix cells are treated as untested and are skipped; only explicit positive or negative tested cells become assay rows.

## PhageHostLearn Benchmark

The 2024 PhageHostLearn study provides a Klebsiella phage-host spot-test benchmark with phage genomes, bacterial genomes, and experimentally tested interactions. Spot-test positives should be interpreted as initial RBP-receptor interactions rather than confirmed productive replication. The data are deposited in Zenodo record `10.5281/zenodo.11061100`, including `phage_host_interactions.csv`; use the normalizer report to record the exact tested-cell count for the reviewed local file and any filtered subset.

Use this source as a first receptor-layer benchmark for H1/H3, not as evidence that defense/counter-defense explains productive infection. H4 still requires productive-infection, plaque, or EOP outcomes.

## Config

`config/assay_matrix_sources.yaml` contains a `sources` list. Each enabled source may define:

- `source_id`: stable source identifier.
- `matrix_path`: reviewed local CSV/TSV matrix path.
- `delimiter`: `comma`, `tab`, or `auto`.
- `host_id_column`: source host identifier column; `auto_first_column` is supported for matrices with an unnamed first column.
- `phage_id_map` and `host_id_map`: TSV mapping files from source IDs to canonical local IDs.
- `output_path`: canonical assay source export path.
- `report_output`: normalization report path.
- `use_source_ids_as_canonical`: only set true for fixtures or sources whose IDs are already canonical.
- `fail_on_unresolved`: whether unresolved tested cells are blocking errors.
- `study_id`, `panel_id`, `assay_type`, `evidence_tier`, `reference`, and outcome-tier defaults.
- `positive_values`, `negative_values`, and `skip_values`.

## Mapping Files

Mapping files use stable columns:

```text
source_id	canonical_id	review_status	notes
```

Only rows with `review_status` of `reviewed`, `accepted`, or `approved` are used. Pending rows generated by `scripts/create_assay_matrix_mapping_templates.py` are curation hints and are ignored. A source ID mapping to multiple canonical IDs is a blocking error. Tested cells with unmapped host or phage IDs are reported and skipped unless `fail_on_unresolved: true`.

## Outputs

The normalizer writes:

- canonical assay source export with `phage_host_assays.tsv` columns;
- report TSV with matrix counts, unresolved entity warnings, unsupported-value errors, and emitted-row counts.

Writes are transactional. If a source has blocking errors, the existing assay source export is not rewritten.

## Example

```bash
python scripts/normalize_assay_matrix.py \
  --config config/assay_matrix_sources.yaml \
  --only-source phagehostlearn_2024_interaction_matrix \
  --root .
```

After the export is generated, run:

```bash
python scripts/import_phage_host_assays.py \
  --config config/assay_imports.yaml \
  --assays-output results/seed/metadata/phage_host_assays.tsv \
  --relationships-output results/seed/metadata/phage_host_relationships.tsv \
  --report-output results/seed/qc/assay_import_report.tsv \
  --root .
```
